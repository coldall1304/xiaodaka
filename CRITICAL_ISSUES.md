### 问题 1: 新用户显示假数据

**现象**:
- 新注册用户看到"连续打卡7天，完成15个计划"
- 显示假计划：每天背单词30个、晨跑5公里、阅读30分钟
- 统计显示：45总打卡、7连续天数、320积分

**预期**:
- 新用户应显示全0数据
- 无假计划

**原因**:
- Vercel 缓存了旧版本页面
- 或浏览器缓存

**解决方案**:
1. 清除 Vercel 缓存重新部署
2. 确保所有 hooks 正确获取真实 API 数据
3. 移除所有硬编码 mock 数据

### 问题 2: 类别标签过于简化

**原网站**:
- 15+ 个预设类别（语文、数学、英语、物理、化学等）
- 用户可自定义类别
- 下拉选择方式

**我们的实现**:
- 仅4个固定按钮（学习、运动、习惯、其他）
- 无法自定义

**影响**:
- 功能不完整，用户体验差
- 无法满足学科分类需求

**解决方案**:
```typescript
// 改为下拉选择
const categories = [
  '语文', '数学', '英语', '物理', '化学', '生物',
  '历史', '地理', '政治', '道德与法治', '信息技术',
  '运动', '娱乐', '技能', '其他'
]
```
